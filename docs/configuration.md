# Configuration reference

Every `CRDT_CAD_*` environment variable the server reads, its default,
and which deployment mode needs it. Generated by grepping the codebase
for `CRDT_CAD_` and reconciling against this table (Phase 19.4) -- if
you add a new one, add a row here too.

## Persistence and scaling

| Variable | Default | Effect |
|---|---|---|
| `CRDT_CAD_DB_PATH` | `<repo>/data/crdt_cad.db` | SQLite file path (Mode A, single replica). Ignored if `CRDT_CAD_DATABASE_URL` is set. |
| `CRDT_CAD_DATABASE_URL` | unset | Postgres DSN -- opts into `PostgresStore` (Mode B, multi-replica). Needs the `postgres` extra (`pip install crdt-cad[postgres]`). |
| `CRDT_CAD_REDIS_URL` | unset | Redis URL -- opts into cross-process WebSocket fan-out (Mode B). Needs the `redis` extra. Meaningless without `CRDT_CAD_DATABASE_URL` also set -- see `k8s/README.md`. |
| `CRDT_CAD_SNAPSHOT_INTERVAL_SECONDS` | `30` | How often the periodic self-healing `frontier` ping fires per room (only sent if something changed). Not the *only* time a room is persisted -- see the two rows below. |
| `CRDT_CAD_PERSIST_MIN_INTERVAL_SECONDS` | `0.5` | Floor on how often live-edit traffic may trigger a durable snapshot persist per room, with a trailing flush so the newest state still lands within one interval of the last edit. Added after `scripts/load_test.py` showed persist-per-message cost grows with document size and drowns the event loop under load (Phase 19.5). `0` restores persist-on-every-message. Explicit saves, imports, AI generation, and SIGTERM shutdown always persist immediately regardless. |
| `CRDT_CAD_VERSION_CHECKPOINT_INTERVAL_SECONDS` | `300` | How often an immutable version-history checkpoint is appended per room (Phase 17), on top of the always-overwritten "latest" snapshot. |
| `CRDT_CAD_MAX_VERSIONS_PER_ROOM` | `20` | How many version-history checkpoints are kept per room before pruning the oldest. |

## Security (all opt-in -- wide open by default)

| Variable | Default | Effect |
|---|---|---|
| `CRDT_CAD_SECRET` | unset (auth off) | Shared secret for minting room tokens (`/api/auth/token`). **Mandatory for any deployment reachable from the public internet** -- see `docs/deployment.md`. |
| `CRDT_CAD_TOKEN_MAX_AGE_SECONDS` | `86400` (24h) | How long a minted room token stays valid. |
| `CRDT_CAD_CORS_ORIGINS` | unset | Comma-separated explicit CORS origin allowlist. Unset: wide open (`*`) if `CRDT_CAD_SECRET` is unset, locked to same-origin only (`[]`) once it's set. Only read once at process startup -- changing it needs a restart. |
| `CRDT_CAD_TRUST_PROXY_HEADERS` | unset (off) | Trust the last hop of `X-Forwarded-For` for the `/generate` rate limiter's client-IP key, instead of the raw socket peer. **Only set this when a reverse proxy you control is the sole way to reach the process** (Caddy, nginx-ingress) -- otherwise any client can bypass the rate limit by spoofing the header. See `security.client_ip`. |

## Accounts (Part 6, opt-in -- fully inert by default)

| Variable | Default | Effect |
|---|---|---|
| `CRDT_CAD_AUTH_MODE` | `tokens` | `accounts` enables user accounts (magic-link + OAuth sign-in, server-side sessions). The default keeps every account feature inert -- no schema created, sign-in routes 404, zero behavior change. Accounts mode **requires `CRDT_CAD_SECRET`** (magic links are signed with it). |
| `CRDT_CAD_MAGIC_LINK_MAX_AGE_SECONDS` | `900` (15 min) | How long a sign-in link stays valid. |
| `CRDT_CAD_SESSION_MAX_AGE_SECONDS` | `2592000` (30 days) | How long a session cookie stays valid. Sessions are server-side; sign-out deletes the row, not just the cookie. |
| `CRDT_CAD_AUTH_DEV_ECHO` | unset (off) | Dev only: when no SMTP is configured, also return the magic link in the API response. **Never enable on a public deployment** -- it lets any visitor obtain a sign-in link for any address. Without it, a no-SMTP deployment only logs links server-side. |
| `CRDT_CAD_SMTP_HOST` / `_PORT` / `_USER` / `_PASSWORD` / `_FROM` / `_STARTTLS` | unset / `587` / – / – / user@host / `1` | SMTP delivery for magic-link e-mails. Unset host = console-echo mode (see above). |
| `CRDT_CAD_OAUTH_GOOGLE_CLIENT_ID` / `_CLIENT_SECRET` | unset | Enables "Continue with Google" (needs the `accounts` extra: `pip install crdt-cad[accounts]`). Redirect URI: `<base>/api/auth/oauth/google/callback`. |
| `CRDT_CAD_OAUTH_GITHUB_CLIENT_ID` / `_CLIENT_SECRET` | unset | Enables "Continue with GitHub", same pattern. |

## Rate limits and resource ceilings

| Variable | Default | Effect |
|---|---|---|
| `CRDT_CAD_WS_OPS_PER_SECOND` | `200` | Per-connection WebSocket ops/sec (token bucket refill rate). |
| `CRDT_CAD_WS_OPS_BURST` | `400` | Per-connection token bucket capacity (burst above the steady rate). |
| `CRDT_CAD_MAX_OPS_PER_ROOM_PER_MINUTE` | `20000` | Room-wide ops/minute ceiling, independent of per-connection limits (caps total room activity regardless of client count). |
| `CRDT_CAD_MAX_OPS_PER_MESSAGE` | `2000` | Max ops accepted in one WebSocket "ops" message; larger messages are rejected outright. |
| `CRDT_CAD_MAX_WS_MESSAGE_BYTES` | `2000000` (2MB) | Max raw WebSocket frame size before the connection is closed (`WS_CLOSE_MESSAGE_TOO_LARGE`). |
| `CRDT_CAD_MAX_ROOMS_PER_SERVER` | `500` | Max distinct rooms one process will create; further `get_or_create` calls raise `RoomLimitExceeded`. |
| `CRDT_CAD_MAX_CLIENTS_PER_ROOM` | `50` | Max simultaneous WebSocket connections in one room (`WS_CLOSE_TOO_MANY_CLIENTS` beyond this). |
| `CRDT_CAD_GENERATE_PER_MINUTE` | `6` | Per-client-IP steady rate for `/api/mesh/{room}/generate` (AI mesh generation -- real LLM spend/CPU cost per call). |
| `CRDT_CAD_GENERATE_BURST` | `3` | Per-client-IP burst capacity for the same limiter. |
| `CRDT_CAD_GENERATION_TIMEOUT_SECONDS` | `60` | Max time a single AI generation call may run before the request fails with 504. |
| `CRDT_CAD_GENERATION_BATCH_SIZE` | `150` | Ops per batch when committing a freshly generated mesh into a room (bounds any single batch's broadcast size). |

## Deployment plumbing

| Variable | Default | Effect |
|---|---|---|
| `CRDT_CAD_STATIC_DIR` | `<repo>/demo/static` | Where the demo HTML/JS/CSS assets are served from. Set explicitly by the Dockerfile, since a non-editable `pip install .` copies the package into site-packages, where the repo-relative default resolves nowhere useful. |
| `CRDT_CAD_DOMAIN` | `localhost` | Read by `Caddyfile` (not the Python app) -- the domain Caddy requests a certificate for. `localhost` falls back to a local self-signed cert. |

## Test-only (not read by the deployed app)

`CRDT_CAD_TEST_DATABASE_URL` / `CRDT_CAD_TEST_REDIS_URL` (point
`tests/test_postgres_store.py` / `test_redis_fanout.py` at a real
Postgres/Redis) and `CRDT_CAD_E2E_LIVE_SERVER_URL` (point the e2e suite's
`live_server` fixture at an already-running deployment instead of
spawning a local `uvicorn` -- see `tests/e2e/conftest.py`) exist only for
the test suite. Setting them in a real deployment has no effect.
